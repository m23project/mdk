Si l'autre serveur DHCP NE PEUT PAS être désactivé:

Dans ce cas, vous devez ajuster la configuration de l'autre serveur DHCP manuellement et éventuellement installer un serveur TFTP. Suivez le manuel de vos serveurs DHCP et TFTP quant à la configuration à y apporter.
  1. Dans l'interface web de m23, choisissez "gPXE/DHCP" quand vous créez un nouveau poste client sous "Type de boot". Utilisez le même nom de poste client sous "Nom du poste client", que celui utilisé par votre serveur DHCP pour le poste client en question.
  2. Ajustez votre serveur DHCP de sorte qu'il assigne le nom du poste client indiqué dans l'interface web m23 au poste client m23.
  3. Le nom du noyau, transmis par votre serveur DHCP au poste client, doit être pxelinux.0. Le serveur TFTP est le serveur m23.
  4. De plus, vous devez configurer votre serveur DHCP de façon à ce qu'il diffuse les paramètres suivants comme paramètres du noyau au poste client:
    m23clientID="numéro du poste client m23" m23server="Nom de domaine du serveur m23 ou son adresse IP" devfs=nomount vga=normal load_ramdisk=1 prompt_ramdisk=0 ramdisk_size=45000 initrd=initrd.gz root=/dev/ram0 rw
    Remplacez numéro du poste client m23 par le numéro du poste client m23 (que vous pouvez trouver chez "Vue d&apos;ensemble des postes clients" dans le lien à chaque poste client) et le Nom de domaine du serveur m23 ou son adresse IP par le nom de domaine du serveur m23 résolvable par le poste client m23 ou son adresse IP. Il sera éventuellement nécessaire de remplacer le numéro 45000 par la valeur indiquée dans le fichier /m23/inc/kernelRamDisk.inc.
  5. Si malgré cela, l'ajustage de votre serveur DHCP ne fonctionne pas, vous devez installer un serveur TFTP sur l'ordinateur hébergeant votre serveur DHCP. La configuration de base du serveur TFTP peut s'inspirer des fichiers de configuration du répertoire /m23/tftp/.
