Aide: Sauvegarde du serveur

Ce dialogue vous aide à sauvegarder votre serveur m23 automatiquement à intervalles réguliers. Les sauvegardes contiennent la configuration du système d'exploitation et la liste des paquets installés en plus des fichiers nécessaires pour la restauration du serveur m23. En utilisant ces informations, il est possible de régénérer le serveur m23 après une perte de données ou quelconque endommagement dans le dernier état sauvegardé. Les données sauvegardées peuvent être téléchargées sur un serveur externe à votre propre réseau ou sur internet après qu'elles aient été cryptées avec GPG pour être à l'abris d'accès indésirés.

Le dialogue de sauvegarde, chiffrement et téléchargement est partagé en deux parties.

  1. Crypter et téléverser la sauvegarde sur un serveur distant ou externe: Cette partie traite du chiffrement des données à sauvegarder et de la copie sur un serveur tierce.

  2. Paramètres pour cron: Les paramètres pour le démon cron déterminent à quelles échéances le serveur m23 doit être sauvegardé automatiquement.



Crypter et téléverser la sauvegarde sur un serveur distant ou externe

Ici, vous entrez les données requises pour l'utilisation de l'espace de stockage externe désiré et sélectionnez la clé GPG pour le chiffrement des vos sauvegardes.

Remarque concernant les clés GPG

Si la clé GPG que vous souhaitez utiliser pour le chiffrement des données n'est pas dans la liste, cliquez sur «Gérer les clés GPG».

  1. Sélectionnez la méthode de sauvegarde de données dans «Service en ligne de sauvegarde de fichiers». vous avez le choix entre:

    • -: Cette option copie l'archive de sauvegarde uniquement sur le disque dur du serveur m23. Vous devriez utiliser cette option seulement si vous ne souhaitez absolument pas utiliser d'autre serveur pour la sauvegarde de vos archives. Dans le cas d'une perte de données, par exemple par défaillance du disque dur du serveur m23, le serveur m23 ne pourra pas être restauré. Vous pouvez copier les fichiers de sauvegarde, qui se trouvent dans le répertoire /m23/root-only/serverBackups/, manuellement sur un média externe.

    • scp: Au moyen de Secure Copy vous pouvez sauvegarder les archives de sauvegarde sur un quelconque ordinateur accessible par son serveur SSH. Cette option s'impose si vous disposez d'un serveur SSH pouvant accueillir les sauvegardes. Si les données de ce serveur sont également sauvegardées par un cycle de sauvegarde, cette option est une possibilité très sûre pour la protection de votre serveur m23.

    • share-online.biz: Si vous sélectionnez cette option, le serveur m23 va transférer vos archives de sauvegarde chiffrées sur un One-Click-Hoster. Si vous n'avez pas de serveur pour copier l'archive de sauvegarde de votre serveur m23, cette option pourrait vous intéresser. Vos données de sauvegarde seront protégées par le chiffrement contre l'accès non autorisé. Le téléchargement peut durer quelque temps.

  2. Cliquez sur «Changer» après avoir choisi.

  3. Dépendant du service en ligne de sauvegarde de fichiers vous devrez fournir différentes indications afin de pouvoir utiliser ce service.

    • scp

      Pour "Secure Copy" les indications suivantes sont nécessaires:

      • Utilisateur: C'est le nom d'utilisateur avec lequel le serveur m23 doit se connecter sur le serveur SCP pour y sauvegarder le fichiers de sauvegarde.

      • Serveur SCP: Ici, entrez le nom ou l'adresse IP par le(la)quel(le) le serveur m23 se connectera au serveur SCP.

      • Répertoire de sauvegarde du serveur: C'est le répertoire sur le serveur SCP où les archives de sauvegarde seront copiés. Entrez le chemin absolu.

      • Clé GPG: Ici, choisissez la clé GPG avec laquelle vous souhaitez chiffrer les sauvegardes avant de les transférer.

      Remarque

      Prenez soin que la clé SSH du superutilisateur root du serveur m23 qui est contenu dans le fichier /root/.ssh/id_dsa.pub serait entré dans le fichier ~/.ssh/authorized_keys de l'utilisateur indiqué «Utilisateur» sur le serveur SCP pour qu'un transfert des fichiers sans mot de passe soit possible. Si vous disposez de la commande «ssh-copy-id», la copie s'effectuera simplement. Pour ce faire, tapez la commande «ssh-copy-id -i /root/.ssh/id_dsa.pub user@serveur_SCP».

    • share-online.biz

      Si vous avez choisi "share-online.biz", vous devez entrer les valeurs suivantes:

      • Utilisateur: C'est le nom d'utilisateur, avec lequel vous êtes registrés chez share-online.biz.

      • Mot de passe: Entrez le mot de passe correspondant.

      • Veuillez répéter le mot de passe: Resaisissez-le pour confirmation.

      • Clé GPG: Sélectionnez la clé GPG, avec laquelle vous souhaitez chiffrer vos archives de sauvegardes avant le transfert.

      Remarque:

      Pour transférer des archives de sauvegarde, il faut s'enregistrer gratuitement sur www.share-online.biz. Les transferts sur et depuis le serveur sont gratuits. Vous devriez vous assurer que les conditions n'aient pas changées. Si c'était le cas, contactez-nous soushhabermann@pc-kiel.de,s.v.p.

  4. Enfin cliquez sur «Enregistrer» (indépendant de la méthode choisi).



Paramètres cron

Avec le démon cron, des tâches se répètant à intervals fixes, peuvent être exécutées. Les éléments de ce dialogue vous aident à fixer l'heure pour l'exécution. Lorsqu'il y a des dates, vous les verrez dans le tableau avec le titre «Entrées cron». Les tâches dans la file d'attente sont marquées d'un point blanc, les tâches qui sont en cours d'exécution avec un point orange.

Processus étape par étape pour ajouter de nouvelles tâches

  1. Sélectionnez d'abord l'unité de temps qu'il vous faut pour la tâche sous «Unité de base pour cron»

  2. Ensuite, cliquez sur «Changer», pour l'ajustement de l'élément dans le dialogue suivant.

  3. Dépendamment de l'unité nécessaire, entrez vos données:

    • Heure: La tâche sera démarrée chaque heure ou toutes les N heures à la minute indiquée.

      • Minute du démarrage: La minute de démarrage de la tâche.

      • Répéter toutes les N heures: L'intervalle entre les exécutions en heures (p.ex. 2 pour toutes les deux heures).

    • Jour: La tâche sera démarrée chaque jour à l'heure indiquée.

      • Minute du démarrage: La minute de démarrage de la tâche.

      • Heure du démarrage: L'heure de démarrage de la tâche.

    • Semaine: La tâche sera démarrée une fois par semaine à l'heure indiquée.

      • Minute du démarrage: La minute de démarrage de la tâche.

      • Heure du démarrage: L'heure de démarrage de la tâche.

      • Jour de la semaine: Le jour de la semaine de démarrage de la tâche.

  4. Enfin, cliquez sur «Ajouter» pour ajouter la tâche.

Effacer une tâche

Vous pouvez effacer une tâche de la liste «Entrées cron» en cliquant sur «Effacer» derrière la tâche concernée.
