Aide: Créer l'image

Par ce dialogue, vous pouvez créer des fichiers image de partitions ou de disques durs entiers d'un poste client; ultérieurement, vous pourrez utiliser ces images pour l'installation d'autres postes client.

Pour cela, choisissez le format de l'image souhaité, le mode de transfert de l'image et la type de compression de l'image.

Dépendant du format de l'image, il est nécessaire que vous saisissiez des informations additionelles dans le champ «Source de l'image», par exemple quelle partition ou quel disque a été enregistré dans le fichier image.

Choisissez ensuite un nom pour votre fichier et saisissez le dans le champ «Nom de l'image».

Enfin, cliquez sur «Créer l'image».

Vous pouvez accéder une vue d'ensemble de tous les fichiers images chez "Configurations du serveur" → "Administrer les fichiers image", où vous pouvez aussi effacer les fichiers images dont vous n'avez plus besoin.

Si vous désirez utiliser le fichier image que vous avez créé ici pour l'installation d'un poste client, choisissez "Imaging" comme distribution pendant la "Configuration" du poste client.

Attention

Le poste client va redémarrer immédiatement après que vous avez cliqué sur "Créer l'image". Ensuite, la création du fichier image débutera!

Tous les données n'ayant pas été sauvegardées avant la validation sur "Créer l'image" seront perdues!

Les fichiers image

Les fichiers seront enregistrés dans le répertoire /m23/data+scripts/clientImages

sous différents types et compressés dans des formats distincts.

Les noms de fichiers sont toujours composés selon le même principe: <nom de l'image><taille de l'image decompressée en bytes><format de l'image><compression>

L'image sera créée comme suit:

Pour la commande:

  • dd: Toutes les données d'une partition ou d'un lecteur seront enregistrées (block par block).

Pour la compression, les valeurs suivantes sont valides:

  • (aucune extension de compression): Le fichier image n'est pas comprimé.

  • gz: La compression sera exécutée avec le programme gzip.

  • bz2: La compression sera exécutée avec le programme bzip2 (le plus efficient dans la plupart des cas).

Le port de communication

Pour le mode de transfert de l'image, vous devez indiquer le port de connexion devant être utilisée tant du coté du poste client que du côté serveur; ce port ne doit pas être bloqué par un pare-feu, etc.

Si vous souhaitez créer plusieures images en même temps, vous devez choisir des ports différents pour chaque prise d'image en parallèle.
