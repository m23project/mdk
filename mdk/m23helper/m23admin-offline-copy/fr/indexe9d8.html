Aide: Ajouter un poste client

(les champs de saisie marquées avec * doivent être remplis correctement dans tous les cas)

  • Préférences: Il vous est possible de choisir parmis les configurations déjà enregistrées, vous aurez par la suite la possibilité de les charger ou de les effacer. Si vous souhaitez enregistrer la configuration actuelle, entrez un nom pertinent et cliquez sur «Enregistrer».

  • Langue: Choisissez la langue du poste client m23. Cette langue sera employée pour le paramétrage du clavier, du bureau et de la console.

  • Nom d'authentification sur le système: Compte de connexion de l'utilisateur sur le poste client.

  • Nom du poste client: Nom unique du poste client ne devant être employé qu'une seule fois dans m23

  • Bureau: Cette information est facultative, vous pouvez indiquer l'endroit où le poste client est localisé (par ex. le numéro et/ou l'étage de la pièce, etc )

  • Prénom: Prénom de l'utilisateur, correspond au nom d'entrée dans le système

  • Nom de famille: Nom de famille de l'utilisateur

  • eMail: Adresse de courrier électronique de l'utilisateur

  • Type de boot: Type de démarrage des postes clients par le réseau.

    Amorçage avec Clé USB ou image ISO

    Si votre client m23 ne peut être démarré par PXE, vous pouvez créer une clé USB amorçable ou graver l'image ISO sur un CD/DVD ou l'utiliser directement dans une machine virtuelle. De plus amples informations sont accessibles sur "Création d'un média d'amorçage" (dans ce menu).

    Note: Serveur DHCP externe

    Si vous souhaitez utiliser m23 avec un serveur DHCP existant, lisez la page suivante: externalDHCP

    Note: UEFI

    Si vous désirez démarrer votre client en utilisant l'UEFI, Vous devez sélectionner "UEFI x86_64". L'architecture sera automatiquement paramétrée pour un système 64 bits, puisque m23 ne supporte l'UEFI que pour les systèmes 64 bits. Si votre client ne supporte pas l'amorçage réseau en mode UEFI, Vous pouvez utiliser un DVD d'amorçage pour architecture 64 bits (Voir l'item "Création d'un média d'amorçage" du menu).

  • Chargeur d'amorçage: Choisissez quel chargeur d'amorçage/boot loader vous souhaitez utiliser pour l'amorçage du noyau Linux et des autres systèmes d'exploitation eventuellement installés.
    Information supplémentaire

    La modification des paramètres de démarrage par des personnes non autorisées n'est rendue possible qu'en saisissant un mot de passe à partir de la version m23 0.6.4 . Il est nécessaire d'entrer le mot des passe de démarrage par le réseau (netboot). Vous pouvez l'extraire dans le centre de contrôle sous «Connexion directe au poste client».

  • Architecture du processeur: Choix d'installation des logiciels en version 32 bits ou 64 bits.

    Renseignement concernant l'architecture du processeur

    m23 est compatible avec les processeurs 32 et 64 bits. Une version 32 bits peut être installée sur un ordinateur 64 bits, l'inverse n'est pas possible. Pour profiter de l'entière potentialité d'un microprocesseur 64 bits, vous devez choisir «amd64». Cette catégorie «amd64» ne comportent pas uniquement les processeurs AMD, mais également ceux d'Intel. Par exemple, les processeurs Intel Pentium D, Pentium Extreme Edition, Celeron D, Core 2 et les processeurs AMD Sempron, Athlon 64, Athlon X2 et Phenom en font partie.

  • Adresse MAC: Adresse MAC de la carte réseau du poste client (par ex. 00D0B723865C)

  • Adresse IP: Adresse IP fixe du poste client (par ex. 192.168.1.23)

  • Masque réseau: Masque de sous-réseau (par ex. 255.255.255.0)

  • Passerelle: Adresse IP de la passerelle (gateway) pour les connexions au réseau internet ou à d'autres connexions externes à votre propre réseau

  • DNS1: Adresse IP du serveur DNS de résolution des noms internet

  • DNS2 (facultatif): Adresse IP du serveur DNS backup

  • Serveur proxy des paquets: Le poste client essaie de télécharger les paquets logiciels sur l'adresse IP du serveur proxy indiqué dans ce champ. En général, il s'agit de l'adresse IP du serveur m23, mais il est également possible d'indiquer un autre serveur proxy. Si vous laissez ce champs de saisie vide, le client téléchargera les paquets logiciels sur internet. Entrez également le numéro du port du serveur proxy. Dans le cas du serveur m23, c'est le port 2323. Lorsque vous ouvrez la boîte de dialogue «Ajouter un client», le serveur m23 est le serveur proxy par défaut.

  • Groupe (facultatif): Nom du groupe de postes client auquel appartient ce poste client

  • Mot de passe de l'utilisateur: Mot de passe de l'utilisateur employé à la première connexion au système. Ce mot de passe comporte six chiffres sélectionnés au hasard, vous pouvez également choisir un mot de passe de votre choix.

  • Mot de passe du superutilisateur «root»: Mot de passe employé pour l'accès au poste client en tant que super-utilisateur (root) . Il comporte huit chiffres sélectionnés au hasard, vous pouvez également choisir un mot de passe de votre choix.

  • Fuseau horaire: Définition du fuseau horaire d'appartenance du poste client.

  • Déterminer l'heure système par NTP: Option de synchronisation de l'heure du poste client par un serveur de temps NTP accessible sur internet.

  • Installer les pilotes de l'imprimante puis détecter l'imprimante: Détection puis installation automatique des imprimantes et de leurs pilotes.

  • Enregistrer les données d'ouverture de session locales sur le poste client.: Cochez cette option si vous désirez un compte local sur le poste client pour l'utilisateur.

  • Ne pas utiliser le LDAP: Pour utiliser uniquement l'authentification locale sur les postes client, cochez «Ne pas utiliser le LDAP».

  • Lire les données d'ouverture de session depuis le serveur LDAP sélectionné.: / Lecture des données d'identification UCS: Pour utiliser les données d'authentification du compte utilisateur enregistrées sur un serveur LDAP ou sur un serveur UCS, cliquez respectivement sur «Lire les données d'ouverture de session depuis le serveur LDAP sélectionné.» ou "Lecture des données d'identification UCS"..

  • Enregistrer les données d'ouverture de session sur le serveur LDAP. / Sauvegarde des données d'identification sur UCS: Enregistrement des données de l' utilisateur renseignées ci-dessus, dans la base de données du serveur LDAP m23 ou le serveur UCS, pour l'authentification sur les postes client, cochez «Enregistrer les données d'ouverture de session sur le serveur LDAP.» ou respectivement "Sauvegarde des données d'identification sur UCS".. Le bon fonctionnement de cette option nécessite un accès complet au serveur LDAP pour l'enregistrement des données sur celui-ci.

  • ID de l'utilisateur, ID du groupe: De plus, vous devez renseigner l'«ID de l'utilisateur» et l'«ID du groupe» pour le compte utilisateur. Ceci est très important; surtout lors de l'utilisation de NFS pour les répertoires personnels.

  • Nom du serveur LDAP: Sélectionnez un serveur LDAP dans la liste «Nom du serveur LDAP». Si le serveur LDAP souhaité n'y apparaissait pas, vous pouvez l'ajouter en cliquant sur «Administrer le serveur LDAP». Si m23 a été installé sur un serveur UCS, cette dernière option est invalide. Le service LDAP du serveur UCS sera utilisé à la place.

  • Répertoire personnel sur le serveur NFS: Ce champ assure l'enregistrement des données du répertoire personnel de l'utilisateur sur un serveur NFS central, afin qu'elles puissent être accédées de tout poste client (en combination avec LDAP). Pour activer le NFS, entrez le nom de l'ordinateur ou son adresse IP et le chemin pour l'enregistrement des répertoires personnels, comme par exemple: 192.168.1.23:/nfs-homes. Si m23 fonctionne sur un serveur UCS, cette dernière option est invalide.

Après avoir vérifié vos entrées, cliquez sur «Ajouter».

Information supplémentaire:

Après l'ajout du nouveau client, ce dernier démarrera sur le réseau (netboot) pour effectuer une séquence de détection du matériel, durant laquelle les informations collectées, comme par ex. la taille du disque dur utilisé, seront envoyées au serveur.

Après l'achèvement du transfert des données l'état du client passera à «jaune» vous pourrez dès lors ajuster les paramètres du client. Pour cela, cliquez sur «Postes client» puis sur «Configuration» dans le menu.
