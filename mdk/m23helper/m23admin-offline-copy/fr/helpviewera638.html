Aide: Utiliser m23 conjointement avec un serveur DHCP externe

Supposons qu'un serveur DHCP diffuse, sur votre réseau local, les paramètres du réseau (IP, masque de réseau, etc ...) pour tous les ordinateurs et qu'il existe un serveur m23 hébergeant également un serveur DHCP, à partir duquel les postes client doivent être installés.

Problème: L'installation des postes client m23 ne fonctionnera pas, parce que le serveur DHCP externe diffuse les paramètres du réseau plus vite que le serveur DHCP m23.

La raison: Le serveur m23 doit assigner l'adresse IP et les autres paramètres du réseau au poste client lors de l'installation du système d'exploitation. Si un autre serveur DHCP fixe ces paramètres à la place du serveur DHCP m23, le poste client ne trouvera pas le serveur m23, en conséquence de quoi, l'image d'amorçage du réseau ne sera pas accessible et le poste client ne pourra pas être installé. Il n'est en principe pas possible de modifier ce comportement, parce que les standards réseaux et les protocoles d'amorçage par le réseau (netboot) suivent une procédure standardisé.

Si le serveur DHCP externe peut être désactivé:

Solutions:

  • Désactivez la réponse du serveur DHCP externe pour les adresses MAC des postes client à installer (au moins) pendant la durée de l'installation.

  • Alternative: Reconfigurez le serveur DHCP externe de sorte qu'il diffuse les paramètres du réseau 2 secondes plus tard, afin que le serveur DHCP m23 réponde toujours plus rapidement.

  • Alternative: Désactivez le serveur DHCP externe pour la durée de l'installation du système d'exploitation des postes m23.

Si l'autre serveur DHCP NE PEUT PAS être désactivé:

Dans ce cas, vous devez ajuster la configuration du serveur DHCP externe manuellement et éventuellement installer un serveur TFTP. Suivez le manuel de vos serveurs DHCP et TFTP quant à la configuration à y apporter.

Alternativement: Création d'un média d'amorçage

Lorsque la modification de la configuration de votre serveur DHCP n'est pas possible, vous pouvez toujours démarrer votre client m23 à partir d'une clé USB ou d'un CD/DVD, puis depuis le shell du client, exécuter le script "fetchJobManual" pour démarrer le processus d'installation du client. De plus amples informations concernant la création du média d'amorçage sont disponibles sur "Création d'un média d'amorçage" dans ce menu.

Configuration dynamique du serveur DHCP

Chaque fois l'état de démarrage par réseau d'un poste client est changé (activé / désactivé), le script "/m23/bin/externalDHCPControl.sh" sera exécuté. Vous pouvez adapter le script selon vos besoins afin de configurer dynamiquement un serveur DHCP existant. Trouvez plus d'information dans les commentaired dans le fichier script.

Serveur DHCP ISC

Si le serveur DHCP externe est un serveur DHCP "ISC", vous pouvez utiliser ce dernier à la place du serveur DHCP m23.


  1. Choisissez "PXE" dans "Type de boot" lorsque vous ajoutez un nouveau client m23 par l'interface web. Saisissez le nom de l'ordinateur à assigner au client m23 par le serveur DHCP dans "Nom du poste client".

  2. Désactivez le serveur DHCP de votre serveur m23: En éditant le fichier suivant 
    /etc/default/isc-dhcp-server
    sur le serveur m23, commentez la ligne ci-dessous en ajoutant un "#" au début de celle-ci 
    INTERFACES=...
    cette dernière deviendra 
    #INTERFACES=...
    puis ajoutez une nouvelle ligne 
    INTERFACES=""
    en dessous de celle précédemment commenté.
    Finalement, redémarrez le serveur DHCP en tapant la commande 
    /etc/init.d/isc-dhcp-server restart
    Le serveur DHCP sera toujours fonctionnel mais ne répondra plus à aucune requète DHCP.

  3. Dans la configuration de votre serveur DHCP, activez l'option 
    use-host-decl-names on
    pour le groupe contenant les clients m23.

  4. Mettez régulièrement à jour le groupe m23 dans la configuration de votre serveur DHCP (par exemple en utilisant une tâche "cron" ou une autre méthode automatisée) utilisant les données de votre serveur DHCP m23 par l'URL 
    http://<m23-Server-IP>/exportDHCPSettings.php
    Cette URL fournit la configuration des clients m23, devant booter sur le réseau, dans le format attendu par le "serveur DHCP ISC". Le groupe pour les clients m23 pourrait par exemple ressembler au code ci-dessous.

    group {

    use-host-decl-names on;

    <A partir des données de exportDHCPSettings.php insérées>

    }

Autre serveur DHCP


  1. Choisissez "PXE" dans "Type de boot" lorsque vous ajoutez un nouveau client m23 par l'interface web. Saisissez le nom de l'ordinateur à assigner au client m23 par le serveur DHCP dans "Nom du poste client".

  2. Ajustez votre serveur DHCP de sorte qu'il assigne le nom du poste client indiqué dans l'interface web m23 au poste client m23.

  3. Le nom du noyau PXE, transmis par votre serveur DHCP au poste client, doit être pxelinux.0. Le serveur TFTP est le serveur m23.

  4. Configurez le serveur DHCP de manière à ce qu'il fournisse l'adresse IP du serveur m23 comme adresse d'origine pour les clients m23.

  5. Si malgré cela, votre serveur DHCP ne fonctionne toujours pas après l'ajustement des paramètres, vous devez installer un serveur TFTP sur l'ordinateur hébergeant votre serveur DHCP. La configuration de base du serveur TFTP peut s'inspirer des fichiers de configuration du répertoire /m23/tftp/.

Proxy DHCP

Il vous sera probablement utile d'utiliser un Proxy DHCP.
