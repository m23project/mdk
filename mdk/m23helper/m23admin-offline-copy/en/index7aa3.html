Help: Manage LDAP servers

You can manage different LDAP servers in this dialog that can be used for user authentification on the clients. An OpenLDAP server can be installed and configured in addition.

  • Add a new LDAP server: Please enter the needed information in the edit lines. The name of the server is "free form" and is entered under "LDAP server name". It is not required to be the same as the hostname of the LDAP server."LDAP server host" has to be the IP or hostname of the LDAP server and must be resolvable from all clients. The preselected port can be cachanged under "LDAP server port". If you enter a valid administrator password under "Password" and "Password (repeated)" the LDAP server can be used for storing new usr accounts. If you leave empty both passwords the server is assumed as read only. Click on "Add" after entering all required data.

  • Change an existing LDAP server: Choose a LDAP server from the drop down list and click on "Load". Now you can see all server data in the edit lines. Modify any values you want and click on "Save changes" afterwards.

  • Delete a LDAP server: Choose a LDAP server from the drop down list and click on "Delete".

  • Edit entries of the LDAP servers: m23 uses the frontend phpLDAPadmin for editing entries. phpLDAPadmin can be started with a click on the phpLDAPadmin logo.
